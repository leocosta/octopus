## Translation conventions

- **Code identifiers, CLI flags, file paths, URLs, component names, prop names,
  environment variable names**: always keep in English exactly as written.
- **Code block contents**: never modified, never translated — even comments
  inside code blocks stay as the source had them.
- **Frontmatter keys**: never translated. Values for `title` and `description`
  are translated; all other keys (`sidebar`, `order`, `tableOfContents`, etc.)
  keep their original values.
- **Register**: informal but professional pt-BR. Use "você", not "tu". Avoid
  Brazilian regionalisms.
- **Anglicisms**: when a term is widely used untranslated in the Brazilian dev
  community (e.g., "deploy", "build", "commit", "merge", "pull request", "branch"),
  keep the English form rather than forcing a stilted Portuguese equivalent.
