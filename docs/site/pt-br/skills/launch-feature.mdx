`launch-feature` pega uma feature que acabou de ser lançada e
produz os artefatos que o time precisa para anunciá-la: uma
seção de landing page, um template de email, posts sociais para
as plataformas que o time usa, uma entrada de changelog comercial
(voz diferente do CHANGELOG técnico) e, opcionalmente, um roteiro
de vídeo e assets de imagem.

## Por que uma skill de lançamento em vez de "o marketer escreve"

Dois motivos:

- **O contexto da feature é local.** A copy de lançamento que
  funciona bem se refere ao problema real que a feature resolve
  — a motivação do RM, as restrições da spec, o diff do PR. Esse
  contexto vive no repo, não na cabeça do marketer. Gerar a
  partir da fonte faz com que o lançamento combine com a feature.
- **O lançamento é multi-canal.** Email, Slack, X, LinkedIn,
  landing page — cada um tem uma voz e um comprimento diferentes.
  Escrever seis versões manualmente é trabalhoso e propenso a
  divergência entre canais. A skill produz todas as seis a partir
  de uma única fonte, aplicando a voz específica de cada canal.

## O que entra em um kit de lançamento

A saída fica em
`docs/marketing/launches/YYYY-MM-DD-<slug>/`:

| Arquivo | Canal |
|---|---|
| `landing.html` | Seção de landing page de marketing |
| `email.md` | Email de newsletter / anúncio |
| `slack.md` | Post no Slack para o canal de clientes / comunidade do time |
| `x.md` | Thread no X / Twitter |
| `linkedin.md` | Post no LinkedIn |
| `changelog.md` | Entrada de changelog comercial (voltada ao usuário) |
| `script.md` | Roteiro de vídeo de 60-90 segundos (se aplicável) |
| `assets/` | Assets de imagem opcionais gerados em sintonia |

O time escolhe quais canais são relevantes e ignora os demais —
a skill produz todos para eliminar a fricção do "eu também
deveria escrever um post no Slack".

## Voz por canal

- **Email** — segunda pessoa, endereça a dor do cliente;
  termina com a chamada para ação.
- **Slack** — curto, casual, com link para a landing page;
  gancho de uma linha + bullet + link.
- **X** — thread; o primeiro post é o gancho, os seguintes
  desenvolvem o caso de uso; o último post é o link.
- **LinkedIn** — mais longo que o X, tom profissional, post único.
- **Seção de landing page** — terceira pessoa, headline focada em
  benefício, pontos técnicos de prova.
- **Changelog comercial** — tom neutro, focado no comportamento
  visível ao usuário, sem jargão interno.

## Aprovação obrigatória por design

A skill rascunha; ela não publica. Todo artefato é entregue como
um arquivo que o time revisa antes de enviar. A allowlist `tools:`
do role marketer explicitamente proíbe as ferramentas de
modificação de arquivos ou de rede que permitiriam a um agent
publicar de forma autônoma — uma proteção deliberada.

## Combinando com `launch-release`

`launch-feature` é para uma feature lançada. `launch-release` é
para uma release inteira que agrupou várias. A variante de
release adiciona um resumo de release notes, uma entrada de
status page e um banner in-app — canais que cabem em uma release
versionada, mas não em uma feature isolada.
