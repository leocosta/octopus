`delegate` roteia uma task pra role certa do Octopus — ou encadeia várias roles
num pipeline — e traz os resultados de volta inline, cada um rotulado com quem o
produziu. É como você obtém a resposta de um especialista (architect, security,
dba) sem sair da conversa.

## O que resolve

O julgamento sênior de um time vive nas suas roles, mas alcançar a certa no meio
da task é atrito: você para, invoca uma role, copia contexto, volta. E quando uma
task precisa de *várias* perspectivas em ordem — desenhar, depois revisar, depois
checar a camada de dados — fazer isso à mão é tedioso e fácil de pular. `delegate`
remove o atrito pra que pedir o especialista certo (ou uma sequência deles) seja
um gesto só.

## Como resolve

- **Uma task pra uma role** — `@architect: …` entrega a task pra essa role e
  retorna a resposta inline, atribuída.
- **Um pipeline de roles** — encadeie roles numa mensagem (ex.: design → review →
  data) e o `delegate` as roda em ordem, passando o output de cada pra próxima.
- **Dispara naturalmente** — uma menção `@<role>:`, uma mensagem encadeada
  multi-role (português ou inglês), ou o comando explícito `/octopus:delegate`.

Todo resultado carrega **atribuição**, então você sempre sabe qual role disse o quê.

## Quando usar

Quando uma task quer a expertise de uma role específica, ou uma sequência de
roles, sem subir uma sessão separada pra cada. Pra orquestração paralela e
duradoura entre muitos agentes, o orquestrador `octopus control` / `run` é a
ferramenta mais pesada.
