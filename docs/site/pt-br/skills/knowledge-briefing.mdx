Uma base de conhecimento só fala quando falam com ela. `knowledge-hygiene` e
`knowledge-synthesize` respondem quando você invoca — mas nada te diz *o que
mudou* ou *o que precisa de você hoje* por conta própria. `knowledge-briefing` é
a parte que vem até você: um resumo curto e ancorado, em cadência, pra que o
blocker registrado ontem não fique invisível até alguém lembrar de perguntar.

## O que resolve

A falha cara de uma base é "você não sabia que precisava olhar." Em algumas áreas
— seis repos, uma dúzia de contextos — ninguém tem o quadro inteiro na cabeça, e
o que precisava de atenção ficou sem ser lido porque checar cada canto à mão não
é trabalho de ninguém. Um briefing inverte isso: a base relata o próprio estado,
em vez de esperar ser consultada área por área.

## Como resolve

O core determinístico (`octopus briefing`) faz o trabalho mecânico; a skill
narra.

- Um **watermark por root** (guardado de forma privada, nunca num repo de time)
  registra quando você olhou pela última vez, então cada run é um verdadeiro
  *desde a última vez que você viu* — não uma janela fixa que você tem que
  lembrar de passar.
- O **delta de mudança** — o sinal próprio do briefing — é o conjunto de nós
  atualizados desde esse watermark.
- Pra seção "precisa de você" ele **compõe os motores irmãos** em vez de
  recomputar: os avisos do `octopus hygiene` (vencido, obsoleto, quebrado) e, no
  `--weekly`, as conexões novas do `octopus synthesize`.
- Toda linha é **ancorada** — cita o nó-fonte de onde veio e não inventa nada. A
  narração roda no **tier de modelo mais barato**, porque transformar um delta em
  prosa é trabalho leve, não raciocínio.

`--daily` avança o watermark (pra que o próximo run continue de onde este parou);
`--weekly` é um rollup só de janela que deixa o watermark intacto. A cadência em
si é hospedada por um scheduler — a skill é um run.

## Uso & parâmetros

```
octopus briefing [--root <id>] [--daily | --weekly] [--since <janela>]
```

| Flag | O que faz | Default |
|---|---|---|
| `--root <id>` | Briefing sobre um knowledge root. | todo root resolvido |
| `--daily` | Briefing de atenção; avança o watermark. | é o padrão |
| `--weekly` | Rollup narrado numa janela de 7 dias; deixa o watermark intacto. | — |
| `--since <janela>` | Sobrescreve a janela pra um run (ex.: `"3 days"`) sem tocar o watermark. | o watermark guardado |
