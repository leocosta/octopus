`enforce-ide` ajusta as convenções do projeto no **editor** — a camada mais
cedo, entre uma tecla e um commit. Quando o editor já formata do jeito que o
projeto espera, a maioria das violações nunca chega a ser digitada.

## O que resolve

O fix mais barato é o que nunca precisa acontecer. Se o editor de cada um indenta,
aspas e trim de forma diferente, o portão de pre-commit e o CI gastam tempo pegando
ruído de estilo, e diffs enchem de churn de reformatação. Alinhar o editor remove
toda essa classe de atrito na origem: a convenção é aplicada enquanto você digita,
não sinalizada três camadas depois.

## Como resolve

- **`.editorconfig` base** — as configs cross-editor (indentação, fim de linha,
  trailing whitespace) que todo editor moderno honra.
- **Config VS Code opt-in** — `.vscode/settings.json` e `.vscode/extensions.json`
  alinhados com o formatter e linter escolhidos do projeto, pra que format-on-save
  e as extensões certas sejam o default.
- **Ciente da stack e respeitoso** — detecta a stack por extensão de arquivo e
  configs existentes, e mescla chaves **de forma conservadora — nunca sobrescrevendo
  um valor que você mesmo definiu**.

É a camada de editor da defesa-em-profundidade: faz par com `enforce-precommit`
(git) e o bundle `guardrails` (o loop do agente), pra que as mesmas convenções
valham em loop, git, CI e IDE.

## Quando usar

Montando ou padronizando um repo, especialmente um de time onde a deriva de editor
causa diffs ruidosos. Use junto com `enforce-precommit` pra defesa-em-profundidade
completa.
