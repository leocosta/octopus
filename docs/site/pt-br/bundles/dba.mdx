`dba` é o bundle para times cuja dor de produção vem da **camada
de dados**: full scans em tabelas sem índice, migrations que
travaram uma tabela quente às 3 da manhã, políticas de retenção
que nunca existiram, chaves Redis sem TTL que encheram a memória,
shard keys Mongo que transformaram um node em todo o cluster.

O bundle adiciona um **segundo gate de approval pré-merge** — a
role `dba` — que roda em paralelo ao `architect` em qualquer PR
que toca a camada de dados. Dois approvals independentes em PRs
de DB, não um.

## O que inclui

```yaml
skills:
  - dba-mssql       # heurísticas SQL Server
  - dba-postgres    # heurísticas PostgreSQL
  - dba-mongodb     # heurísticas MongoDB
  - dba-redis       # heurísticas Redis
roles:
  - dba             # revisor especialista, roda ao lado do architect
```
