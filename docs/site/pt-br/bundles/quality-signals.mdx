`quality-signals` é o subconjunto signal-only do
[`quality`](/octopus/bundles/quality/). Todas as skills aqui são advisory por
design: elas surfacam divergências, sinalizam trabalho incompleto e propõem
melhorias, mas nenhuma delas bloqueia um merge. A camada bloqueante está no
[`quality-audits`](/octopus/bundles/quality-audits/).

## Por que isso existe

Nem todo time está pronto para — ou precisa de — gates bloqueantes desde o
início. Um time migrando um repo existente, ou que já roda suas próprias
checagens mandatórias de CI, pode querer que o Octopus atue como um segundo par
de olhos, não como um gatekeeper. Essas skills se encaixam exatamente nesse
papel. Também são úteis em repos onde o `quality-audits` já está instalado, mas
o time quer que a superfície advisory seja visualmente distinta da bloqueante.

## O que inclui

```yaml
skills:
  - audit-grounding    # divergência da fonte de verdade (CONTEXT.md, knowledge/)
  - audit-verification # confirma que todos os critérios de aceite foram atendidos
  - audit-style        # design-rules, over-engineering, padrões de legibilidade
  - audit-config       # rules obsoletas, skills fantasmas, paths de hook deprecados
  - refactor-deepen    # detecção de módulos rasos; propõe aprofundamento via deletion test
roles: []
```

[Fonte: `bundles/quality-signals.yml`](https://github.com/leocosta/octopus/blob/main/bundles/quality-signals.yml)

### Por que cada uma está aqui

- **`audit-grounding`** — captura o modo de falha semântico que as auditorias de
  domínio não veem: um agent afirmando uma convenção do time ou fato de domínio
  que contradiz a fonte de verdade (`CONTEXT.md`, `knowledge/`). Seu Stop hook
  dispara ao fim de cada tarefa; a skill reporta divergências e deixa a decisão
  com o humano.
- **`audit-verification`** — verifica que o trabalho de fato satisfaz os critérios
  de aceite antes de encerrar a sessão. Previne a falha silenciosa de "pronto mas
  incompleto", onde o agent marca uma tarefa como concluída antes de cobrir todos
  os casos especificados.
- **`audit-style`** — inspeciona violações de design-rules (over-engineering,
  abstração prematura, valores mágicos, módulos god) e problemas de legibilidade
  que não chegam ao nível de um achado de security ou money-logic. Orienta em
  direção ao vocabulário canônico do módulo.
- **`audit-config`** — captura drift de configuração: rules calibradas para
  modelos mais antigos, skills fantasmas referenciadas em bundles, paths de hook
  obsoletos, assinaturas de comando deprecadas. Recomendada trimestralmente;
  nunca bloqueia um merge.
- **`refactor-deepen`** — encontra módulos rasos e propõe oportunidades concretas
  de aprofundamento usando o deletion test e canonicalização de vocabulário. É
  trabalho arquitetural advisory, não um gate.

### Por que algumas coisas não estão aqui

- **`audit-all`, `audit-contracts`, `architect`, `security`** — são a camada de
  gate bloqueante e pertencem ao
  [`quality-audits`](/octopus/bundles/quality-audits/).
- **`knowledge-hygiene`, `knowledge-synthesize`, `knowledge-briefing`** — são
  operações do knowledge loop e vivem no
  [`knowledge-ops`](/octopus/bundles/knowledge-ops/).

## Workflow que isso habilita

Essas skills rodam em pausas naturais, não como gates rígidos:

- **`audit-grounding`** dispara automaticamente via Stop hook ao final de cada
  tarefa com diff. Você revisa o relatório de divergências; nenhum merge é
  bloqueado.
- **`audit-verification`** roda quando você (ou o agent) está prestes a encerrar
  uma tarefa. Verifica os critérios de aceite e surfaça qualquer caso não coberto.
- **`audit-style`** e **`refactor-deepen`** são rodadas sob demanda — geralmente
  antes de um PR ou depois que uma feature está funcionalmente completa — para
  capturar problemas de design antes da revisão.
- **`audit-config`** é uma rotina de housekeeping periódica (trimestral) para
  manter a própria configuração do Octopus alinhada com o comportamento atual
  dos modelos.

## Compõe com

- **[starter](/octopus/bundles/starter/)** — fundação obrigatória.
- **[quality-audits](/octopus/bundles/quality-audits/)** — o complemento
  bloqueante; juntos cobrem toda a superfície do `quality`.
- **[quality](/octopus/bundles/quality/)** — se quiser tudo em uma única
  instalação, use o `quality` diretamente.
