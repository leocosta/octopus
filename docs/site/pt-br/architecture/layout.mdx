O repo do Octopus é o catálogo de primitivas mais a
maquinaria de geração que as entrega. A árvore de arquivos é a
arquitetura — cada diretório de nível superior é uma categoria
de conteúdo configurável.

## Lendo a árvore

Três regras tornam a árvore navegável:

- **Um diretório por primitiva.** `skills/`, `commands/`, `hooks/`,
  `roles/`, `rules/`, `bundles/`. A categoria está no path.
- **Um subdiretório por instância.** Cada skill é `skills/<name>/`
  com seu próprio `SKILL.md`. Cada bundle é um YAML. Cada hook é
  um shell script sob o diretório de fase correto. O nome da
  instância está no path.
- **Sem arquivos cruzando categorias.** Uma skill nunca mora em
  `commands/`, um hook nunca mora em `skills/`. A fronteira da
  categoria é imposta por onde o arquivo está, não pelo seu conteúdo.

## O que mora fora da árvore (nos projetos dos usuários)

O repo do Octopus é o catálogo. Projetos de usuário consomem o
catálogo referenciando-o a partir do seu `.octopus.yml`. Após
rodar `octopus setup`, o projeto do usuário recebe:

```
user-project/
├── .octopus.yml                            # The only file the user edits
├── .octopus/                               # Runtime state (gitignored)
├── .claude/CLAUDE.md                       # Generated
├── .claude/settings.json                   # Generated
├── .claude/rules/                          # Generated (symlinks to Octopus)
├── .claude/skills/                         # Generated (symlinks to Octopus)
├── .claude/commands/                       # Generated (symlinks to Octopus)
├── .claude/agents/                         # Generated (roles, symlinked)
├── .github/copilot-instructions.md         # Generated
├── AGENTS.md                               # Generated for Codex
├── GEMINI.md                               # Generated for Gemini
└── .opencode/                              # Generated for OpenCode
```

Todos os arquivos gerados estão no gitignore — a fonte da verdade
é o `.octopus.yml` mais a versão do catálogo do Octopus. Regere
a qualquer momento com `octopus setup`.
