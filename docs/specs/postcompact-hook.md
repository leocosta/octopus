## Problem Statement

Octopus has a `PreCompact` hook (`hooks/pre-compact/save-state.sh`) that saves state before context compaction. However, there is no `PostCompact` hook to re-inject critical context after compaction occurs.

Result: after a compaction, Claude loses critical instructions that were in the context (e.g. working branches, in-progress tasks, session-specific rules) and the saved `pre-compact.yml` is never read back.

## Goals

1. Create script `hooks/post-compact/reload-context.sh` that reads the saved state and re-injects it via `echo` to the hook's stdout (injected back into Claude's context)
2. Register the `PostCompact` hook in `hooks/hooks.json`
3. The hook must be non-destructive: if there is no saved state, it does not fail and injects nothing

## Non-Goals

- Do not re-implement save-state (already handled by PreCompact)
- Do not re-inject the full contents of CLAUDE.md (Claude already has access to it)
- Do not create new persistence infrastructure — reuse `~/.octopus/state/pre-compact.yml`

## Design

### Overview

Claude Code supports the `PostCompact` event. A hook registered for this event can inject text into the context via stdout. The script will read the `pre-compact.yml` file generated by the existing hook and format the content as a brief reminder.

### Detailed Design

**Script: `hooks/post-compact/reload-context.sh`**

```bash
#!/usr/bin/env bash
# Re-inject working state after context compaction

set -euo pipefail

state_file="${HOME}/.octopus/state/pre-compact.yml"

if [[ ! -f "$state_file" ]]; then
  exit 0
fi

# Read state and emit as context reminder
branch=$(grep '^branch:' "$state_file" | cut -d' ' -f2- | tr -d '\n')
timestamp=$(grep '^timestamp:' "$state_file" | cut -d' ' -f2- | tr -d '\n')

dirty=$(awk '/^dirty_files:/{flag=1; next} /^[a-z]/{flag=0} flag && /^  -/{print}' "$state_file")
staged=$(awk '/^staged_files:/{flag=1; next} /^[a-z]/{flag=0} flag && /^  -/{print}' "$state_file")

# Only output if there's meaningful state to restore
if [[ -z "$branch" && -z "$dirty" && -z "$staged" ]]; then
  exit 0
fi

echo "## Context Restored After Compaction"
echo ""
[[ -n "$branch" ]] && echo "**Branch:** $branch"
[[ -n "$timestamp" ]] && echo "**State saved at:** $timestamp"
if [[ -n "$dirty" ]]; then
  echo ""
  echo "**Uncommitted changes:**"
  echo "$dirty"
fi
if [[ -n "$staged" ]]; then
  echo ""
  echo "**Staged files:**"
  echo "$staged"
fi
```

**Registration in `hooks/hooks.json`** — add after the `PreCompact` block:

```json
"PostCompact": [
  {
    "matcher": "",
    "hooks": [
      {
        "type": "command",
        "command": "octopus/hooks/post-compact/reload-context.sh",
        "id": "reload-context",
        "timeout": 10000
      }
    ]
  }
]
```

**Directory structure**:
```
hooks/
  pre-compact/
    save-state.sh      ← already exists
  post-compact/
    reload-context.sh  ← new
```

### Migration / Backward Compatibility

- Purely additive. No existing behavior is changed.
- If `~/.octopus/state/pre-compact.yml` does not exist, the script exits silently with `exit 0`.
- The `PostCompact` hook is ignored by versions of Claude Code that do not support it.

## Agent Support

| Agent | PostCompact equivalent | Notes |
|---|---|---|
| **Claude Code** | `PostCompact` | Native lifecycle event, documented Mar 2026 (Boris Cherny, Part 7). Fires after context compression; stdout is injected back into context. |
| **OpenCode** | None | The OpenCode plugin SDK (v7.1.9) exposes `experimental.session.compacting` (pre-compact only) and the observable `session.compacted` event, but no post-compact hook that can inject text into context. Context re-injection via stdout is not supported. |
| **Copilot** | None | No native hook system. |
| **Codex** | None | No native hook system. |
| **Antigravity** | None | No native hook system. |

**Conclusion:** `PostCompact` is Claude Code-only. The `hooks/post-compact/reload-context.sh` script will be registered exclusively under Claude Code's `hooks.json` and has no OpenCode equivalent.

## Implementation Plan

2. **`hooks/hooks.json`** — add `PostCompact` block after the `PreCompact` block (line ~90)

**Constraints**:
- Pure bash — no external dependencies beyond coreutils (grep, awk, cut)
- The script must be idempotent and never fail (non-zero exits break the hook flow)
- Output via `stdout` — that is what Claude Code injects back into the context
- Do not inject redundant content: if branch and diffs are empty, `exit 0` silently

## Testing Strategy

- `test_postcompact_with_state`: state file exists with branch and diffs → stdout contains restored context section
- `test_postcompact_no_state`: state file does not exist → empty stdout, exit 0
- `test_postcompact_empty_state`: state file exists but branch and diffs are empty → empty stdout, exit 0
- Verify that `hooks.json` contains `PostCompact` after `setup.sh` runs (integration test)

## Risks

- None significant. The hook is read-only and non-destructive.
- Minor risk: if the format of `pre-compact.yml` changes in the future, the parsing may break. Mitigation: both scripts (save and reload) are co-located and easy to update together.
