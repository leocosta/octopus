Open a pull request. The agent writes the body; the CLI handles the
mechanics (branch push + `gh pr create`).

## Arguments

Use `$ARGUMENTS` to determine the target branch.

- Target branch: `$1` (default: `main`)

## Instructions

1. **Resolve target branch.** If `$1` is empty, run
   `git branch -r | grep -v HEAD` and ask the user which branch to
   target (default: `main`).

2. **Resolve the PR-body language** by consulting, in order:
   - `.octopus.yml` → `language.code` (per-scope form).
   - `.octopus.yml` → `language:` (short form, applies to all scopes).
   - Cascade from `rules/common/language.md` (docs/, `git log`, locales/).
   - Default: English.
   Remember the resolved value — you will apply it to every line of
   the body you write.

3. **Collect context.** Let `BASE` be the target branch.
   - Commits: `git log ${BASE}..HEAD --format='%H%n%s%n%b%n---%n'`.
   - File stats: `git diff --stat ${BASE}..HEAD`.
   - Semantic diff: `git diff ${BASE}..HEAD` — scan for the *meaning*
     of the change, not line-by-line.
   - Cross-references: note any `RM-NNN`, `docs/specs/*`, or
     `docs/adr/*` paths mentioned in commit messages; include them
     in the body where relevant.

4. **Read the template.** Open `cli/pr-body-default.md`. Follow its
   section contract and the hard rules in its header comment.

5. **Write the body to a tempfile.**
   - Create a file with `mktemp --suffix=.md` (store the path).
   - The body has exactly three required sections: `## What`,
     `## Why` (omit if truly self-evident), `## Test plan`.
   - Collapse the file list inside a `<details>` block with a
     `<summary>Files changed (<N>)</summary>`.
   - End with `<sub>🐙 generated by Octopus</sub>` on its own line,
     preceded by a `---` horizontal rule.
   - Write every line in the language resolved at step 2. Do not
     mix languages. Do not default to the conversation language.
   - Never dump `git log` as a bullet list.
   - Never list files outside the `<details>` block.

6. **Open the PR** by running:
   ```
   octopus pr-open --target <branch> --body-file <tempfile>
   ```
   The CLI aborts if `--body-file` is missing — always pass it.

7. **Capture the PR number** from the output line
   `OCTOPUS_PR=<number>`.

8. **Show the submitted body.** Run:
   ```
   gh pr view <number> --json url,body -q '"URL: \(.url)\n\n## PR Body\n\(.body)"'
   ```
   Display the full URL and body so the user can see exactly what
   was submitted.

9. **Close out.** Suggest `/octopus:pr-review <number>` as the next
   step.
